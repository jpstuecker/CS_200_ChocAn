Written Solution

In this project, we designed a class diagram which showcased our skills in designing a program in which cohesion and coupling are major factors in its conception. The goal of designing this class diagram was to build our program with the lowest coupling and the highest cohesion possible. The challenge was to design these classes in a way that had the fewest interactions possible so as to lower the dependency required for each class to operate as intended. We think we did well in this aspect as the majority of our classes only interact with only one or two other classes with our highest coupled class interacting with three classes. All of our classes have been carefully and intentionally planned out so that every part has a functional purpose in completing the various tasks of the program. With the ideas of coupling and cohesion in mind, we think we have designed a highly maintainable and readable program which should help us in the future with constructing the program.
